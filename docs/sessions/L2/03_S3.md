## üíª **Part 1: What Are Functions?**

### **The Problem: Repetitive Code**

Imagine you need to greet users multiple times in your program:

```python
# Without functions - repetitive code!
name1 = "Alice"
print(f"Hello, {name1}! Welcome to Python.")
print(f"Have a great day, {name1}!")

name2 = "Bob"
print(f"Hello, {Bob}! Welcome to Python.")
print(f"Have a great day, {Bob}!")

name3 = "Charlie"
print(f"Hello, {Charlie}! Welcome to Python.")
print(f"Have a great day, {Charlie}!")
```

This is repetitive and hard to maintain. If you want to change the greeting, you'd have to update it in multiple places!

### **The Solution: Functions**

A **function** is a reusable block of code that performs a specific task. Think of it as a recipe you can use over and over.

**Benefits:**

- ‚úÖ **Reusability**: Write once, use many times
- ‚úÖ **Organization**: Break complex problems into smaller pieces
- ‚úÖ **Maintainability**: Change code in one place, affects everywhere
- ‚úÖ **Readability**: Code becomes self-documenting

### **Built-in Functions You Already Know**

You've been using functions since Level 1:

```python
print("Hello")        # print() is a function
len([1, 2, 3])        # len() is a function
input("Enter name:")  # input() is a function
type(42)              # type() is a function
```

Now you'll learn to create your own!

## üíª **Part 2: Defining Functions**

### **Basic Function Syntax**

You define a function using the `def` keyword:

```python
def function_name():
    # Code to execute
    pass
```

**Breaking it down:**

- `def`: Keyword that says "I'm defining a function"
- `function_name`: Name of your function (follow naming conventions)
- `()`: Parameters go here (we'll cover this in Session 5)
- `:`: Indicates the function body starts
- Indented code: What the function does

### **Your First Function**

Let's create a simple greeting function:

```python
def greet():
    print("Hello! Welcome to Python.")
    print("Have a great day!")

# Now you can use it!
greet()  # Calls the function
greet()  # Call it again - no need to rewrite code!
greet()  # And again!
```

**Output:**

```text
Hello! Welcome to Python.
Have a great day!
Hello! Welcome to Python.
Have a great day!
Hello! Welcome to Python.
Have a great day!
```

### **Functions with Code Logic**

Functions can contain any code:

```python
def show_menu():
    print("=" * 30)
    print("1. View items")
    print("2. Add item")
    print("3. Delete item")
    print("4. Exit")
    print("=" * 30)

# Use it whenever you need the menu
show_menu()
```

## üíª **Part 3: Calling Functions**

### **How Function Calls Work**

When you call a function, Python:

1. Jumps to the function definition
2. Executes all code in the function
3. Returns to where the function was called
4. Continues with the next line

```python
print("Before function")  # Line 1

def my_function():
    print("Inside function")  # Line 3
    print("Still inside")      # Line 4

print("After definition")  # Line 2
my_function()              # Line 5 - calls function
print("After function")    # Line 6
```

**Output:**

```text
Before function
After definition
Inside function
Still inside
After function
```

**Key Point:** The function definition doesn't run until you call it!

### **Calling Functions Multiple Times**

The power of functions: write once, use many times:

```python
def print_separator():
    print("-" * 40)

print_separator()
print("Section 1: Introduction")
print_separator()
print("Section 2: Main Content")
print_separator()
print("Section 3: Conclusion")
print_separator()
```

## üíª **Part 4: Why Functions Matter - DRY Principle**

### **DRY: Don't Repeat Yourself**

**DRY** is a fundamental programming principle: **Don't Repeat Yourself**.

**Without functions (WET - Write Everything Twice):**

```python
# Calculate area of rectangle 1
length1 = 5
width1 = 3
area1 = length1 * width1
print(f"Area: {area1}")

# Calculate area of rectangle 2
length2 = 7
width2 = 4
area2 = length2 * width2
print(f"Area: {area2}")

# Calculate area of rectangle 3
length3 = 10
width3 = 2
area3 = length3 * width3
print(f"Area: {area3}")
```

**With functions (DRY):**

```python
def calculate_area(length, width):
    area = length * width
    print(f"Area: {area}")

# Now just call it!
calculate_area(5, 3)
calculate_area(7, 4)
calculate_area(10, 2)
```

**Benefits:**

- ‚úÖ Less code to write
- ‚úÖ Easier to maintain (change once, works everywhere)
- ‚úÖ Fewer bugs (test once, use everywhere)
- ‚úÖ More readable

## üíª **Part 5: Function Naming Conventions**

### **Python Naming Rules**

Follow these rules for function names:

**‚úÖ Good Function Names:**

```python
def greet_user():          # Clear, descriptive
def calculate_total():     # Verb + noun
def is_valid():            # Boolean functions start with 'is'
def get_name():            # 'get' for retrieving data
def process_data():        # 'process' for transformations
```

**‚ùå Bad Function Names:**

```python
def x():                   # Too short, unclear
def function1():            # Not descriptive
def do_stuff():            # Vague
def CalculateTotal():      # Should be lowercase (PascalCase is for classes)
```

### **Naming Best Practices**

1. **Use lowercase** with underscores: `calculate_total()`
2. **Be descriptive**: `calculate_total()` not `calc()`
3. **Use verbs**: Functions do things, so use action words
4. **Keep it concise**: Not too long, but clear

## üìù **Key Takeaways**

- **Functions** are reusable blocks of code that perform specific tasks
- **Define functions** with `def function_name():`
- **Call functions** by writing their name followed by `()`
- **DRY principle**: Don't Repeat Yourself - use functions to avoid duplication
- **Function names** should be descriptive, lowercase, with underscores
- **Organize code** by breaking problems into smaller functions

## üîß **Troubleshooting**

### **Common Issues:**

**Issue:** `NameError: name 'my_function' is not defined`  
**Solution:** Make sure you define the function before calling it, or the function name is spelled correctly

**Issue:** Function doesn't run  
**Solution:** Remember to call the function with `()` - just writing `function_name` doesn't execute it

**Issue:** IndentationError  
**Solution:** All code inside a function must be indented (4 spaces or 1 tab)
