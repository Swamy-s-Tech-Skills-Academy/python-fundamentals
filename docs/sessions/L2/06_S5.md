## üíª **Part 1: Function Parameters**

### **The Problem: Functions Without Input**

In Session 3, we created functions without parameters:

```python
def greet():
    print("Hello! Welcome to Python.")
```

This function always says the same thing. What if we want to greet different people?

### **The Solution: Parameters**

**Parameters** let you pass data into functions, making them flexible and reusable:

```python
def greet(name):
    print(f"Hello, {name}! Welcome to Python.")

# Now we can greet anyone!
greet("Alice")   # "Hello, Alice! Welcome to Python."
greet("Bob")     # "Hello, Bob! Welcome to Python."
greet("Charlie") # "Hello, Charlie! Welcome to Python."
```

**Benefits:**

- ‚úÖ Same function, different results
- ‚úÖ More flexible and reusable
- ‚úÖ Less code duplication

### **How Parameters Work**

```python
def calculate_area(length, width):
    area = length * width
    print(f"Area: {area}")

# Call with two values
calculate_area(5, 3)  # Area: 15
calculate_area(10, 4)  # Area: 40
```

**What happens:**

1. Function receives `length = 5` and `width = 3`
2. Calculates `area = 5 * 3 = 15`
3. Prints the result

## üíª **Part 2: Parameter Types**

### **Positional Parameters (Default)**

Parameters are **positional** by default - order matters:

```python
def introduce(name, age, city):
    print(f"Hi! I'm {name}, {age} years old, from {city}.")

# Order matters!
introduce("Alice", 25, "New York")  # Correct order
introduce("New York", "Alice", 25)  # Wrong! Confusing output
```

### **Keyword Parameters**

You can specify parameters by name:

```python
def introduce(name, age, city):
    print(f"Hi! I'm {name}, {age} years old, from {city}.")

# Using keyword arguments (order doesn't matter!)
introduce(name="Alice", age=25, city="New York")
introduce(city="New York", name="Alice", age=25)  # Same result!
```

**Benefits:**

- ‚úÖ Order doesn't matter
- ‚úÖ More readable
- ‚úÖ Less error-prone

### **Default Parameters**

You can provide default values:

```python
def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

# Use default
greet("Alice")              # "Hello, Alice!"

# Override default
greet("Bob", "Hi")          # "Hi, Bob!"
greet("Charlie", "Good morning")  # "Good morning, Charlie!"
```

**Rules:**

- Parameters with defaults must come **after** parameters without defaults
- You can mix positional, keyword, and default parameters

```python
# Good: default comes last
def calculate(price, discount=0.1):
    return price * (1 - discount)

# Bad: default comes first (syntax error!)
# def calculate(discount=0.1, price):  # Error!
```

## üíª **Part 3: Return Values**

### **Functions That Return Data**

So far, our functions only printed. But functions can **return** data:

```python
def calculate_area(length, width):
    area = length * width
    return area  # Send the result back

# Use the returned value
result = calculate_area(5, 3)
print(f"Area: {result}")  # Area: 15

# Or use it directly
print(f"Area: {calculate_area(10, 4)}")  # Area: 40
```

### **Why Return Instead of Print?**

**Returning is better because:**

- ‚úÖ Function can be used in calculations
- ‚úÖ More flexible (caller decides what to do)
- ‚úÖ Can be tested more easily

```python
# With return - flexible
def calculate_area(length, width):
    return length * width

area1 = calculate_area(5, 3)
area2 = calculate_area(10, 4)
total = area1 + area2  # Can use in calculations!
print(f"Total area: {total}")

# With print - less flexible
def calculate_area_print(length, width):
    print(length * width)

# Can't easily use the result in calculations
```

### **Functions Without Return**

If a function doesn't have `return`, it returns `None`:

```python
def greet(name):
    print(f"Hello, {name}!")

result = greet("Alice")
print(result)  # None (function doesn't return anything)
```

## üíª **Part 4: Multiple Return Values**

### **Returning Multiple Values with Tuples**

Functions can return multiple values using tuples:

```python
def get_name_age():
    name = "Alice"
    age = 25
    return name, age  # Returns a tuple

# Unpack the tuple
name, age = get_name_age()
print(f"{name} is {age} years old")  # "Alice is 25 years old"
```

### **Practical Example**

```python
def calculate_stats(numbers):
    if not numbers:
        return None, None, None
    
    total = sum(numbers)
    count = len(numbers)
    average = total / count
    
    return total, count, average  # Return multiple values

scores = [85, 90, 92, 88, 95]
total, count, average = calculate_stats(scores)
print(f"Total: {total}, Count: {count}, Average: {average:.2f}")
```

### **Returning Structured Data**

You can return tuples with meaningful structure:

```python
def get_student_info():
    name = "Alice"
    age = 25
    grade = "A"
    return (name, age, grade)  # Explicit tuple

name, age, grade = get_student_info()
print(f"{name} (age {age}) has grade {grade}")
```

## üíª **Part 5: None and Documentation**

### **Understanding None**

`None` is Python's way of saying "nothing" or "no value":

```python
def greet(name):
    print(f"Hello, {name}!")

result = greet("Alice")
if result is None:
    print("Function didn't return anything")
```

**When functions return None:**

- Function has no `return` statement
- Function has `return` with no value: `return`
- Function explicitly returns `None`: `return None`

### **Function Documentation: Docstrings**

**Docstrings** document what your function does:

```python
def calculate_area(length, width):
    """
    Calculate the area of a rectangle.
    
    Parameters:
    - length: The length of the rectangle
    - width: The width of the rectangle
    
    Returns:
    - The area of the rectangle (length * width)
    """
    return length * width
```

**Benefits:**
- ‚úÖ Explains what the function does
- ‚úÖ Documents parameters and return values
- ‚úÖ Accessible via `help()` function
- ‚úÖ Makes code self-documenting

**Accessing Docstrings:**

```python
help(calculate_area)  # Shows the docstring
```

## üìù **Key Takeaways**

- **Parameters** pass data into functions (positional, keyword, default)
- **Return values** send data back from functions
- **Multiple returns** use tuples: `return value1, value2`
- **None** is returned when a function has no return statement
- **Docstrings** document functions for clarity and help
- **Default parameters** make functions more flexible

---

## üîß **Troubleshooting**

### **Common Issues:**

**Issue:** `TypeError: missing required positional argument`  
**Solution:** Make sure you provide all required parameters (those without defaults)

**Issue:** Default parameters before required parameters  
**Solution:** Parameters with defaults must come after those without defaults

**Issue:** Can't use return value  
**Solution:** Make sure your function has a `return` statement
