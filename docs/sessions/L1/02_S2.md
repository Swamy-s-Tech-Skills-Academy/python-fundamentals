## ğŸ’» **Part 1: Variables & Naming**

### **What is a Variable?**

A **variable** is like a labeled box where you store information. You give it a name, and Python remembers what's inside.

Think of it like this:

- **Variable name** = Label on the box
- **Variable value** = What's inside the box

### **Creating Variables (Assignment)**

In Python, you create variables using the **assignment operator** (`=`):

```python
name = "Alice"
age = 25
```

**How to read this:**

- `name = "Alice"` means "store the value 'Alice' in a variable called 'name'"
- `age = 25` means "store the value 25 in a variable called 'age'"

### **Variable Naming Rules**

Python has specific rules for naming variables:

âœ… **Allowed:**

- Letters (a-z, A-Z)
- Numbers (0-9) - but NOT at the start
- Underscores (_)
- Case-sensitive (Age and age are different)

âŒ **Not Allowed:**

- Spaces
- Special characters (@, #, $, %, etc.)
- Starting with a number
- Python keywords (like `print`, `if`, `for`, etc.)

**Good Variable Names:**

```python
first_name = "Alice"
age_in_years = 25
is_student = True
total_score = 95.5
```

**Bad Variable Names:**

```python
first name = "Alice"    # âŒ Space not allowed
2nd_place = "Bob"       # âŒ Can't start with number
print = "Hello"         # âŒ 'print' is a Python keyword
```

### **Naming Conventions (Best Practices)**

While Python allows many names, following conventions makes code readable:

1. **Use descriptive names**: `age` is better than `a`
2. **Use lowercase with underscores**: `first_name` not `FirstName`
3. **Be consistent**: If you use `user_name`, don't switch to `username` later
4. **Avoid abbreviations**: `age` is better than `ag`

**Examples:**

```python
# âœ… Good naming
student_name = "Alice"
student_age = 20
is_enrolled = True

# âŒ Less clear
sn = "Alice"
sa = 20
ie = True
```

## ğŸ“Š **Part 2: Data Types Introduction**

Python has four **fundamental data types** you'll use constantly:

### **1. Integers (`int`)**

Whole numbers (positive, negative, or zero):

```python
age = 25
temperature = -10
count = 0
population = 8000000000
```

### **2. Floating-Point Numbers (`float`)**

Numbers with decimal points:

```python
height = 5.9
price = 19.99
temperature = -12.304
pi = 3.14159
```

**Note:** Even if you write `5.0`, Python treats it as a float, not an integer.

### **3. Strings (`str`)**

Text data (always in quotes):

```python
name = "Alice"
message = 'Hello, World!'
favorite_color = "blue"
```

**String Rules:**

- Use single quotes `'text'` or double quotes `"text"`
- Both work the same way
- Be consistent in your code

### **4. Booleans (`bool`)**

True or False values (exactly two options):

```python
is_student = True
is_graduated = False
has_license = True
```

**Important:**

- `True` and `False` must start with capital letters
- No quotes around them (they're not strings!)

### **Quick Reference Table**

| Data Type | Example Values | Use For |
| --------- | -------------- | ------- |
| `int` | `25`, `-10`, `0`, `1000` | Whole numbers |
| `float` | `3.14`, `-5.5`, `0.0`, `99.9` | Decimal numbers |
| `str` | `"Hello"`, `'Python'`, `"123"` | Text data |
| `bool` | `True`, `False` | Yes/No, On/Off |

## ğŸ”„ **Part 3: Dynamic Typing**

### **What is Dynamic Typing?**

Python is **dynamically typed** - this means Python figures out the data type automatically based on what you assign.

**You don't need to declare the type!**

```python
# Python automatically knows:
name = "Alice"        # This is a string (str)
age = 25              # This is an integer (int)
height = 5.9          # This is a float
is_student = True     # This is a boolean (bool)
```

**Compare to other languages:**

- Some languages require: `int age = 25;` (you must say it's an int)
- Python just needs: `age = 25` (Python figures it out)

### **Variables Can Change Types**

In Python, a variable can hold different types at different times:

```python
value = 25           # value is an int
value = "Hello"      # Now value is a str
value = 3.14        # Now value is a float
```

**This is allowed in Python!** The variable name stays the same, but what's inside can change type.

### **Why This Matters**

Dynamic typing makes Python:

- âœ… **Easier to learn** - Less to remember
- âœ… **More flexible** - Change types as needed
- âœ… **Faster to write** - No type declarations

**But be careful:** Make sure you know what type your variable is when you use it!

## ğŸ” **Part 4: Type Checking**

### **Using `type()` Function**

You learned `type()` in Session 1. Now let's use it to check data types:

```python
name = "Alice"
age = 25
height = 5.9
is_student = True

print(type(name))        # <class 'str'>
print(type(age))         # <class 'int'>
print(type(height))      # <class 'float'>
print(type(is_student))  # <class 'bool'>
```

### **Using `isinstance()` Function**

`isinstance()` checks if a value is a specific type (returns `True` or `False`):

```python
age = 25

print(isinstance(age, int))    # True
print(isinstance(age, str))    # False
print(isinstance(age, float))  # False
```

**Why use `isinstance()`?**

- More readable than `type() ==`
- Better for checking types in conditions
- Recommended by Python style guides

**Example:**

```python
value = 25

if isinstance(value, int):
    print("This is an integer!")
else:
    print("This is not an integer!")
```

## ğŸ”„ **Part 5: Type Conversion**

### **What is Type Conversion?**

**Type conversion** (also called **casting**) means changing a value from one data type to another.

### **Converting to Integer (`int()`)**

```python
# Convert string to integer
age_str = "25"
age_int = int(age_str)      # Now it's 25 (int), not "25" (str)

# Convert float to integer (truncates decimal)
price = 19.99
price_int = int(price)      # Becomes 19 (drops .99)
```

**Important:** The string must contain a valid number!

```python
int("25")      # âœ… Works: 25
int("25.5")    # âŒ Error: can't convert "25.5" directly to int
int("hello")   # âŒ Error: "hello" is not a number
```

### **Converting to Float (`float()`)**

```python
# Convert string to float
price_str = "19.99"
price_float = float(price_str)  # Now it's 19.99 (float)

# Convert integer to float
age = 25
age_float = float(age)          # Becomes 25.0 (float)
```

### **Converting to String (`str()`)**

```python
# Convert number to string
age = 25
age_str = str(age)           # Becomes "25" (string)

price = 19.99
price_str = str(price)       # Becomes "19.99" (string)
```

**Why convert to string?**

- To combine numbers with other pieces of text (`"Hello " + 9` will fail in REPL)
- To display numbers inside user-friendly messages

```python
age = 25

# Using str() when doing manual concatenation
message = "I am " + str(age) + " years old"
print(message)  # I am 25 years old

# Using an f-string (recommended)
message = f"I am {age} years old"
print(message)
```

### **Converting to Boolean (`bool()`)**

```python
# Most values convert to True
bool(1)        # True
bool(25)       # True
bool(-25)       # True
bool("hello")  # True
bool(3.14)     # True
bool(" ")       # True 

# Only these convert to False:
bool(0)        # False
bool(0.0)      # False
bool("")       # False (empty string)
bool(False)    # False
bool(None)     # False
```

### **Common Conversion Patterns**

#### **Pattern 1: User Input (Always Strings)**

```python
# input() always returns a string!
age_str = input("Enter your age: ")  # User types: 25
age_int = int(age_str)              # Convert to integer
```

#### **Pattern 2: Combining Numbers and Text**

```python
age = 25
message = "I am " + str(age) + " years old"
# Or use f-strings (easier!):
message = f"I am {age} years old"
```

#### **Pattern 3: Calculations with User Input**

```python
# Get numbers from user
num1_str = input("Enter first number: ")
num2_str = input("Enter second number: ")

# Convert to integers for calculation
num1 = int(num1_str)
num2 = int(num2_str)

# Now we can do math
result = num1 + num2
print(f"The sum is {result}")
```

## ğŸ”§ **Troubleshooting**

### **Common Issues:**

#### **"NameError: name 'variable' is not defined"**

**Problem:** You're trying to use a variable before creating it.

```python
# âŒ Wrong - using variable before creating it
print(name)
name = "Alice"

# âœ… Correct - create variable first
name = "Alice"
print(name)
```

#### **"ValueError: invalid literal for int()"**

**Problem:** Trying to convert a string that isn't a valid number.

```python
# âŒ Wrong - "hello" can't be converted to int
age = int("hello")

# âœ… Correct - use a valid number string
age = int("25")
```

#### **"TypeError: can only concatenate str (not 'int') to str"**

**Problem:** Trying to combine string and number without conversion.

```text
>>> "Hello " + 9
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: can only concatenate str (not "int") to str
```

```python
# âŒ Wrong - can't add string and int directly
age = 25
message = "I am " + age + " years old"

# âœ… Correct - convert number to string
message = "I am " + str(age) + " years old"

# âœ… Or use f-strings (easier!)
message = f"I am {age} years old"
```

#### **Variable name errors**

**Problem:** Using invalid characters or Python keywords.

```python
# âŒ Wrong - space in variable name
first name = "Alice"

# âŒ Wrong - starting with number
2nd_place = "Bob"

# âŒ Wrong - using Python keyword
print = "Hello"

# âœ… Correct - use valid names
first_name = "Alice"
second_place = "Bob"
message = "Hello"
```
